외부 진입점

진입점은 둘뿐: Web UI(HTTPS, 브라우저)와 Frontend :7233(gRPC + mTLS). 외부 SDK worker·CLI는 Frontend로 접속한다.

서버 4-role

frontend(stateless 게이트웨이) · history(shard·상태) · matching(task queue) · worker service(서버 내부용). Web UI는 별도.

namespace 분리

server · db · worker가 각각 별도 k8s namespace. worker ns를 지워도 server·db는 안전. SDK worker는 서버 밖 프로세스(배포만 이 repo).

플랫폼 공통 · 교체 가능

ArgoCD · cert-manager · ESO · Vault · cnpg · Prometheus. 사용 환경에 맞춰 교체/제거 가능. docs/platform-assumptions.md.

인증서 조달 (mTLS)

cert-manager (in-cluster) · Vault (외부 worker)

  1. CAVault PKI가 Root → Intermediate 공통 CA를 보유.
  2. 발급in-cluster는 cert-manager가 ClusterIssuer로 leaf 발급·자동 갱신 → Secret.
  3. 마운트Secret을 파드에 디렉토리 마운트 → TEMPORAL_TLS_*_PATH가 읽음.
  4. 외부클러스터 밖 worker는 Vault approle로 직접 발급(cert-manager 미도달).

시크릿 조달 (ESO)

External Secrets Operator

  1. seed운영자가 Vault KV에 값을 시드(repo엔 참조만, 비밀값 없음).
  2. syncESO가 ClusterSecretStore로 KV → k8s Secret 동기화.
  3. inject파드가 envFrom으로 주입받음.
  4. rotaterefreshInterval 주기로 갱신(반영엔 파드 재기동 필요).

⚠️ cert-manager는 Secret만 갱신하고 파드를 재기동하지 않는다. 앱이 cert를 시작 시 1회만 읽으면 갱신이 반영 안 됨 → hot-reload 또는 Reloader로 롤링 재기동 필요.
